# VibeShelf — Product Requirements Document

**바이브코더를 위한 큐레이티드 오픈소스 템플릿 마켓플레이스**

v3.0 Final | 2026.04

---

## 1. Problem

바이브코딩 도구(Cursor, Lovable, Bolt 등)의 확산으로 비개발자도 앱을 만들 수 있게 되었다. 그러나 매번 빈 프롬프트에서 시작해야 하고, GitHub에 이미 존재하는 수백만 개의 오픈소스 프로젝트를 활용하지 못하고 있다.

GitHub에는 좋은 보일러플레이트와 스타터 킷이 넘치지만, 바이브코더에게는 세 가지 벽이 있다:

- **발견의 벽:** 뭘 검색해야 하는지 모르고, 검색해도 수만 개의 결과에서 좋은 것을 고를 수 없다
- **판단의 벽:** README와 코드만으로는 이 템플릿이 자기 프로젝트에 맞는지 판단할 수 없다
- **적용의 벽:** 포크한 후 어떻게 커스터마이즈해야 하는지 모른다

---

## 2. Solution

VibeShelf는 GitHub의 오픈소스 레포를 자동으로 수집하고, AI로 평가/분류한 뒤, 바이브코더가 시각적으로 탐색하고 바로 사용할 수 있게 큐레이션하는 마켓플레이스다.

### 핵심 가치 제안

- 코드를 읽지 않고 눈으로 보고 고른다 (시각적 프리뷰)
- AI가 품질을 미리 검증해준다 (자동 스코어링)
- 원클릭으로 내 것으로 가져간다 (GitHub 포크)

---

## 3. Target user

**Primary:** 바이브코더 — 코딩 경험이 거의 없지만 Cursor, Lovable, Bolt 등을 활용해 앱을 만들려는 비개발자. 파운더, 디자이너, PM, 마케터 등.

**Secondary:** 인디 개발자 — 매번 보일러플레이트를 처음부터 세팅하는 것이 번거로운 솔로 개발자. 검증된 스타터킷으로 시간을 아끼고 싶은 사람.

---

## 4. Core value engine: 수집 - 평가 - 분류 파이프라인

**이 파이프라인이 VibeShelf의 핵심 경쟁력이다.** UI, 결제, 부가 기능은 경쟁자가 빠르게 복제할 수 있지만, 지속적으로 양질의 레포를 수집하고 정확하게 평가/분류하는 파이프라인은 시간과 데이터가 쌓여야 가능하기 때문에 구조적 해자가 된다. 개발 리소스의 우선순위는 항상 이 파이프라인의 정확도와 커버리지 향상에 둔다.

### 4.1 수집 (Ingestion)

GitHub API를 통해 오픈소스 레포를 자동으로 수집하여 초기 공급 문제(cold start)를 해결한다.

#### 수집 기준

- 라이선스: MIT, Apache 2.0, BSD만 허용 (상업적 사용 가능한 permissive 라이선스)
- 키워드: SaaS boilerplate, starter kit, template, dashboard, landing page 등
- 활성도: 최근 6개월 이내 커밋이 있는 레포만
- 최소 품질: 스타 10개 이상, README 존재

#### 수집 데이터

- 메타데이터: 이름, 설명, 스타/포크 수, 라이선스, 마지막 커밋일
- 기술 스택: package.json, requirements.txt 등에서 자동 감지
- README 전문: AI 분류 및 이미지/GIF 추출용

#### 갱신 주기

| 작업 | 주기 | 목적 |
|---|---|---|
| 신규 레포 발견 | 매일 | 새로운 양질의 레포 유입 |
| 기존 레포 메타 갱신 | 주 1회 | 스타/포크 수, 마지막 커밋 최신화 |
| 비활성 레포 탈락 | 월 1회 | 12개월 이상 커밋 없는 레포 제거 |
| 스코어 재평가 | 월 1회 | 의존성 변경, README 업데이트 반영 |

#### 법적 요건

- 원본 GitHub 링크, 라이선스, 원저자 정보를 항상 명시
- 코드를 자체 호스팅하지 않고 GitHub으로 연결 (포크/리다이렉트)
- GPL 등 copyleft 라이선스 레포는 자동 제외

### 4.2 평가 (Evaluation)

수집된 레포를 Gemini Flash-Lite API로 자동 분석하여 품질 스코어를 산출한다. Gemini Flash-Lite를 사용하는 이유는 비용 효율성이다: 분류/스코어링은 경량 모델로 충분하며, 100만 토큰당 $0.10으로 2,000개 레포 전체 평가 비용이 $0.05 수준이다.

#### 품질 스코어 (5점 만점)

| 항목 | 측정 방법 | 가중치 |
|---|---|---|
| 문서화 | README 길이, 스크린샷/GIF 포함 여부, 설치 가이드 유무 | 25% |
| 유지보수 | 마지막 커밋 시점, 이슈 응답률, 릴리스 빈도 | 25% |
| 인기도 | 스타, 포크, 워치 수 | 20% |
| 코드 건강 | 테스트 존재 여부, 의존성 수 및 최신성 | 20% |
| 바이브코딩 호환 | CSS 변수 사용, .cursorrules 존재, 환경변수 문서화 | 10% |

#### 정확도 개선 루프

AI 스코어링은 초기에 완벽하지 않다. 다음 피드백 루프를 통해 지속적으로 정확도를 높인다:

1. **유저 행동 데이터:** 높은 스코어인데 포크가 적은 레포 → 스코어링 기준 재검토
2. **리뷰 평점 괴리 분석:** 유저 리뷰 평점과 AI 스코어 간 괴리가 큰 레포 → AI 프롬프트 튜닝
3. **수동 큐레이션 오버라이드:** 초기에는 주 1회 수동 검수로 AI 분류 오류를 교정하고, 교정 데이터를 AI 프롬프트 개선에 재투입

### 4.3 분류 (Classification)

README를 기반으로 카테고리와 태그를 자동 배정한다.

#### 카테고리 (앱 유형별)

SaaS / 이커머스 / 대시보드 / 랜딩페이지 / AI 도구 / 유틸리티 / 게임 / 기타

#### 기능 태그 (필터용)

- 기술 스택: Next.js, React, Python, Supabase, Stripe 등
- 바이브코딩 도구 호환: Cursor / Bolt / Lovable / Replit
- 기능: 인증 포함, 결제 연동, 다크모드, 반응형, AI 통합 등

#### 분류 정확도 목표

| 지표 | 런칭 시 | 3개월 후 |
|---|---|---|
| 카테고리 정확도 | 85% | 93%+ |
| 기능 태그 정확도 | 80% | 90%+ |
| 스팸/저품질 필터링 | 90% | 97%+ |

이 정확도 목표를 달성하기 위해, 초기 4주간은 전체 수집 레포의 20%를 수동 검수하여 AI 분류 결과를 교정한다. 이후 월 5%로 줄여가며 AI 자율성을 높인다.

## 5. Features

### 5.1 시각적 프리뷰

바이브코더가 코드를 읽지 않고도 3초 안에 판단할 수 있게 한다. 기존 경쟁 서비스 대비 핵심 차별점이다.

#### 프리뷰 소스 (우선순위 순)

1. **README 내 GIF:** 기능 시연 효과, 카드에서 호버 시 자동 재생
2. **README 내 정적 이미지:** 대표 스크린샷으로 사용
3. **데모 URL 감지 시:** Puppeteer로 자동 스크린샷 생성
4. **프리뷰 없음:** AI가 README 기반 UI 설명 생성, '프리뷰 없음' 뱃지 + 스코어 감점

#### 표시 방식

- 카드 뷰: Pinterest 스타일 이미지 그리드, GIF는 호버 시 자동 재생
- 상세 페이지: 스크린샷 갤러리 (데스크톱 + 모바일), 기술 스택 태그, README 요약
- 카드에 리뷰 수 + 평균 평점 뱃지 표시

### 5.2 탐색 및 필터

- 카테고리 필터 (SaaS, 이커머스, 대시보드 등)
- 기술 스택 필터 (Next.js, React, Python 등)
- 품질 스코어 필터 (3점 이상, 4점 이상 등)
- 기능 태그 필터 (인증, 결제, AI 등)
- 정렬: 인기순, 최신순, 스코어순, 리뷰 평점순

### 5.3 원클릭 포크

GitHub OAuth 연동을 통해 유저의 GitHub 계정으로 원클릭 포크. 코드를 직접 호스팅하지 않으므로 라이선스 리스크가 없다.

### 5.4 유저 리뷰 및 커뮤니티 쇼케이스

실제 사용자가 템플릿 적용 결과물을 공유하여, 다음 유저가 이 템플릿으로 뭘 만들 수 있는지 눈으로 확인할 수 있게 한다.

#### 리뷰 구성

- 평점 (5점 만점)
- 텍스트 후기 (선택)
- 적용 결과 이미지 또는 GIF 첨부 (최대 5장)
- 사용한 바이브코딩 도구 태그 (Cursor / Lovable / Bolt 등)

#### 표시 방식

- 템플릿 상세 페이지에 '커뮤니티 쇼케이스' 섹션으로 노출
- 이미지 첨부 리뷰를 최상단에 우선 배치

#### 작성 조건

- GitHub 계정 연동 유저만 작성 가능 (스팸 방지)
- 해당 템플릿을 포크한 이력이 있는 유저만 작성 가능 (실사용 검증)

리뷰 데이터는 파이프라인 정확도 개선에도 활용된다: 유저 평점과 AI 스코어 간 괴리가 큰 레포를 자동 플래그하여 스코어링 기준을 지속적으로 교정한다.

## 6. Pro features ($15/mo)

### 6.1 브랜드 매칭 프리뷰

유저가 자신의 디자인 정보(Brand Profile)를 한 번 등록하면, 마켓플레이스의 모든 템플릿을 '내 브랜드 버전'으로 미리 볼 수 있다.

#### Brand Profile 입력

- 컬러 피커: primary, secondary, accent, background, foreground (5가지)
- 폰트 선택: Google Fonts 범위 내 드롭다운
- 모서리 스타일: 둥근 / 직각 / 필(pill)
- (선택) 로고 이미지 업로드

#### 적용 방식

템플릿의 CSS 변수(:root의 --primary, --background 등)를 유저의 Brand Profile 값으로 오버라이드하여 iframe 내에서 렌더링. 상세 페이지에서 '원본 ↔ 내 브랜드' 토글로 비교 가능.

#### 제한 사항

- CSS 변수 기반이 아닌 레포는 적용 불가 → '브랜드 매칭 지원' 뱃지로 구분
- 레이아웃/구조 변경은 불포함 — 색상, 폰트, 둥글기만 적용

### 6.2 프롬프트 플레이북

각 템플릿에 대해 AI가 자동 생성하는 커스터마이즈 가이드. 바이브코더가 포크 후 막히지 않도록 돕는다. 프롬프트 플레이북 생성에는 고품질 텍스트가 필요하므로 Claude Sonnet 4.6을 사용한다.

- 템플릿별 구조 요약 (파일 구조, 주요 컴포넌트)
- 커스터마이즈 프롬프트 예시 3~5개
- Cursor / Bolt / Lovable 각 도구별 맞춤 가이드

## 12. Risks and mitigations

| 리스크 | 영향 | 완화 방안 |
|---|---|---|
| AI 분류 정확도 부족 | 잘못된 카테고리 → 유저 이탈 | 수동 검수 20% + 피드백 루프 주 1회 |
| GitHub API rate limit | 수집 속도 제한 | 캐싱 + 점진적 수집 + 인증 토큰 풀 |
